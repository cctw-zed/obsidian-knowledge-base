# 个人知识库

这是一个基于Obsidian的个人知识管理系统，专注于知识沉淀与积累。

## 📁 目录结构

```
obsidian-repo/
├── Daily/              # 日记/每日笔记
├── Notes/              # 核心笔记库（主要工作区）
│   ├── Tech/           # 技术笔记
│   ├── Business/       # 商业思考
│   └── Personal/       # 个人成长
├── Literature/         # 读书/文章/课程笔记
├── Concepts/           # 概念卡片/知识点提炼
├── Clippings/          # 网页摘录/灵感收集
├── MOC/                # Map of Content（知识地图）
├── Templates/          # 笔记模板
└── Attachments/        # 图片/文件附件
```

## 🔄 工作流程

1. **快速记录**：灵感或阅读内容先放入 `Clippings/`
2. **每日整理**：在 `Daily/` 中记录每天的想法和待整理内容
3. **正式笔记**：经过思考的内容写入 `Notes/` 对应领域
4. **阅读笔记**：书籍、文章、课程笔记放入 `Literature/`
5. **概念提炼**：将知识点提炼为原子化概念放入 `Concepts/`
6. **建立地图**：在 `MOC/` 中创建知识地图，连接相关笔记

## 💡 使用建议

### 笔记原则
- **原子化**：每个笔记只讨论一个主题
- **链接优先**：通过 `[[双向链接]]` 建立知识网络，而非依赖文件夹
- **标签辅助**：使用 `#标签` 进行横向分类
- **定期回顾**：每周回顾 Daily 笔记，提炼有价值内容

### 命名规范
- 笔记文件：使用描述性标题，如 `React Hooks 原理解析.md`
- 日记文件：使用日期格式，如 `2026-01-13.md`
- MOC文件：以领域命名，如 `Tech-Map.md`

### 推荐插件
- **Dataview**：动态查询和展示笔记
- **Templater**：高级模板功能
- **Calendar**：日记管理
- **Excalidraw**：手绘图表
- **QuickAdd**：快速创建笔记

## 🎯 核心理念

这个知识库采用 **Kepano 简洁方法**，强调：
1. 扁平化结构，避免深层嵌套
2. 双向链接驱动，建立知识网络
3. MOC 作为导航，串联相关主题
4. 持续积累，形成个人知识体系
